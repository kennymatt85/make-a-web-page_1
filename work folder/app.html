Basics of the Web
  • World Wide Web was invented in the 1990's
  • Now consists of approximately 30 billion pages!
  • The majority of content on the web is made up of Hyper Text Markup Language (HTML for short)
  • Hyperlinks link the web together
So how does the Web work?



For example if I login to my computer and click on Google Chrome (My Browser). I will then type in a url this sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the my computer where Google Chrome interprets the page and displays it my screen. This video provides further information.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML is made up of:

  • Text Content- "What you see"
  • Markup- "What it looks like"
  • References to other documents e.g. Images and Videos
  • Links to other pages

Creating white space

The Languages of Web Development

There are 3 Languages used in front end web development

  1. HTML- structures the document
  2. CSS- styles the document
  3. Javascript- provides interactive components for the document

Tree-like Structure

The tree like structure you can see in step 3 is referred to as the Document Object Model (DOM for short). The browser turns HTML tags into objects within a tree i.e. a parent element can have many child elements. See this video for further details

Boxes, Boxes, Everywhere!

Everything on a webpage is made up of boxes the tag that allows us to do this is the div tag. This tag is a block tag the inline version of this is the span tag. Although you may see several different shapes on a webpage they are still a box with CSS providing style to create the illusion of an irregular shape.

Text Editors for Programmers

In order to write code it is imperative that we use a code editor. Code editors make our life much easier as the automatically generate a closing tag (in most cases) this reduces the chance of mistakes.

Lesson 1.3 Adding CSS Style and HTML Structure

What is CSS?

CSS stands for cascading style sheets. This means a list of rules on the style of the document where the most specific rule applies. The most important thing to remember when designing the CSS sheet is to avoid repetition as this will save time and reduce chance of errors.

Inheritance and CSS

Inheritance is one of the most importance ideas within CSS. This is the idea that styling not only applies to the specified element but all its decendants unless over-ruled by a more specific rule.There are several benefits to this including saving bandwidth,enhancing loading time and saving time coding. However it is important to note that descendant elements do not inherit all properties in general it is only text related ones.

How create your rules using CSS

There are 3 parts to creating a rule in CSS first you must use a selector followed by a property and a value, to reduce repetition you only have to state the selector once you can then list your pairs of properties and values. There are 3 types of selector I will be using tag selector, class selector and id selector. Id selector being the most specific i.e over-rules the other selectors and tag selector being the least specific i.e. easiest to over-rule. When using a tag selector you just reference the tag in CSS, when using a class selector reference the class name preceeded by a dot and when referencing an id selector reference the id name preceeded by a hash tag. There are 3 other types of selector; attribute selectors, positional selectors and other pseudo selectors but as I am still very much a beginner and it is important to master the first 3. Briefly property refers to the style you would like to change and value is what you would like to change it to for example color and red. You can find a full list of attributes and styles here.

How to include CSS styling on my webpage

There are three ways of including CSS styling on my webpage the first I am going to discount as it is generally a bad idea, results in lots of repetition and can lead to a lot of mistakes i.e. writing inline with HTML. The second is ok for small projects but isn't the most organised i.e. writing CSS in the head you can do this by using the style tag and writing your CSS within the tag. This could be particularly useful for eBay descriptions as you are unable to link to an external stylesheet. The third is to write in a separate CSS file this is the most organised and what I have done on this project.
